The LoRA roadmap is:

Start from your pretrained Qwen base model.
This is the backbone.
Keep the original model weights frozen.
Add LoRA adapters to a few transformer layers.
Start with q_proj and v_proj.
These adapters are the trainable part of the policy.
Add a value head on top of the shared backbone.
The LM head still predicts next-token logits.
The value head predicts expected reward for PPO.
Make a separate frozen reference model.
Same original Qwen weights.
No LoRA updates.
Used for the KL penalty during PPO.
So the architecture becomes:

policy model = frozen Qwen backbone + trainable LoRA adapters + value head
reference model = frozen original Qwen model
Keep your verifier fixed.
